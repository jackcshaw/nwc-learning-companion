# The Irreducible Officer Site Design Brief

## Purpose

The website is the public essay presentation and routing layer. It should make
**The Irreducible Officer** readable, downloadable, and easy to connect to the
companion and workbench without embedding those repos inside the site.

## First Viewport

The first signal is the essay itself: title, subtitle, public status, and the
start of the reading experience. The site should not feel like a generic
AI-education landing page.

## Navigation

The final site should give readers clear paths to:

- Essay: the final essay, table of contents, PDF, and visual argument aids.
- Companion: the runnable agent-mode repo and setup prompt.
- Workbench: faculty implementation layer with current templates and future context-curation direction.
- Sources: public evidence map and links for deeper reading.

The scrolling section navigation is a keeper. It gives the long essay a sense
of place, shows progress, and lets readers move through the argument without
turning the page into a conventional landing site.

## Design Direction

The current site and the companion may contain useful design direction, but the
final design does not have to preserve the current style exactly. The target is
a cleaner cross between this essay-reading shell and the Librarian site Jack
liked: serious, architectural, source-aware, and built around a readable long
argument with clear paths into companion practice and faculty workbench use.

## Visual Rhythm

Use visuals sparingly and in service of the argument:

- a purpose / frame / reliance / accountability / transfer standard;
- a failure-mode diagnostic for the Section II list;
- a simple map of the package: essay, companion, workbench, future context layer.

Pull quotes or extra visual treatments should be added only when they make the
argument easier to scan.

## Maturity Language

Keep the project surfaces honest:

- Essay: final public argument.
- Companion: runnable agent-mode source kit.
- Workbench: faculty draft/use-now toolkit.
- Library/context curation: future direction inspired by The Librarian, not a deployed NWC system.

## Public-Safe Boundary

The website repo must not contain private course artifacts, audit transcripts,
Proof scratch files, local drafting files, unpublished materials, or the full
companion/workbench source trees.
